\section{Method}
\subsection{Data Collection}
We take the standard CIE 1931 XYZ color matching functions \cite{1931TrOS...33...73S} as our target spectral response functions for reconstruction.
Further, we run the same analysis on modern XYZ color matching functions, such as those seen in Sharpe et al. \cite{sharpe2005luminous}

First, we draft a sample set of filters by scraping the \cite{rosco_mycolor} website for spectral response functions. These provide a helpful and expansive set of filters. 

Then, for our measured filter set, we use a standard set of Rosco Supergel \cite{rosco_mycolor} filters and measure their spectral response functions using a spectrophotometer under an Ianiro Varibeam 800W bulb.
We construct a dataset spanning $\lambda$ 400-750 at 1nm intervals, and normalize the response functions to a maximum of 1.

\subsection{Formalization}

We minimize for the distance between the target spectral response functions and the reconstructed functions from the selected filters:
\begin{equation}
    \argmin_{\boldsymbol{M},f} \int_{\lambda}  \boldsymbol{L}(\boldsymbol{P}(\lambda),\boldsymbol{G}_{\boldsymbol{M}}(\lambda))d\lambda
    \label{eq:1}
\end{equation}

We consider all color channels equally and decompose target color matching functions $\boldsymbol{P}(\lambda)$ and simulated color matching functions $\boldsymbol{G}(\lambda)$ into $p_c(\lambda)$ and $G_c(\lambda)$, respectively.

\begin{equation}
    \argmin_{\boldsymbol{M},f} \sum_{c \in {X,Y,Z}} \int_{\lambda}  \boldsymbol{L}(p_c(\lambda),G_{c,\boldsymbol{M}}(\lambda))d\lambda
    \label{eq:2}
\end{equation}


Where $G_{c,\boldsymbol{M}}(\lambda)$ is the reconstructed spectral response function for color channel $c$ from the selected filters, which can be expressed as:
\begin{equation}
    G_{c,\boldsymbol{M}}(\lambda) = \sum_{k}^{N} f_k M_{c}I(\lambda)R(\lambda)e_{k}(\lambda) \, d\lambda
    \label{eq:3}
\end{equation}

For approximations of XYZ, we follow the work of Hardeberg \cite{Filterselection} in using RMSE as our loss function, again equally weighted across each channel. 
Additionally, with RMSE loss function, the solution to \ref{eq:1} can be achieved in a computationally efficient manner by computing the pseudoinverse.
\begin{equation}
    \argmin_{\boldsymbol{M}} ||\boldsymbol{P} - \boldsymbol{M}\boldsymbol{G_M}||^2
\end{equation}
For $F$ as Mean Squared Error (MSE), the optimal weights can be found using the pseudo-inverse:
\begin{equation}
    \boldsymbol{M} = \boldsymbol{P}\boldsymbol{G_M}^T(\boldsymbol{G_M}\boldsymbol{G_M}^T)^{-1}
\end{equation}  

And for other $F$, such as SMAPE, we use gradient descent to find the optimal weights, informing our initial guess with the MSE solution.

\subsection{Filter Selection}
\subsubsection{Brute Force Search}
In the naive solution, we find the optimal filter combination for each $N$ by selecting the combination with the lowest loss $F$, considering all combinations of filters.
We plot the errors across all combinations for N=4:

As the search space grows combinatorially with $N$, we cannot feasibly consider all combinations.
We introduce a 'prune' factor which removes filters based on a cosine similarity metric. 

\subsubsection{PCA Analysis}
To find the optimal filter combination for each $N$ without brute force search, we can use PCA analysis to find the optimal weights $M$ for all filters, and then select the top $N$ filters with the highest weights.


\subsubsection{Blended Methods}

While brute force methods guarantee minimum possible loss by investigating the entire search space, large $N$ implementations render this technique impossible. 
Crucially, we can decrease the speed of combinatoric explosion by pruning our set of filters, but the problem of which filter to exclude in a pairwise filter pruning decision is not trivial. 

Similarly, PCA and other heuristic methods generally miss the optimal filter combination. 

To address this problem, we use a blended method which finds an ansatz using PCA or another heuristic method for $N-N2$ filters, and then greedily considers the final $N2$ filters from all remaining. 
This provides a middle ground to the performance-accuracy tradeoff by searching depth and then breadth of the combinatoric tree. 
We see that a significant shortfall grows with $N$ as $N2=0$, but nearly disappears at $N2>0$.
